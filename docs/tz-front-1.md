## 1. Назначение

Создание и ведение семестровых работ: участники, аудитории, рассадка,
печать вариантов, оценки. После проведения ученик видит место, сканы,
оценки. Каждая роль видит свой срез.

---

## 2. Роли и права

Роли: 1 ученик, 2 учитель, 3 оператор, 4 админ.
- Классрук = учитель, привязанный к классу как руководитель.
- Модератор работы = создатель / админ предмета / оператор+.

Бэк проверяет права (403 при отказе → «Нет доступа»). Видимость полей
определяет бэк; вручную ничего не прячем. Текущий пользователь —
`getCurrentUser()` (см. FRONTEND_GUIDE).

## 4. Экраны

### 3.1 Список работ (ученик)

- **Маршрут:** `/student/works`
- **API:** `GET /api/works/` — ученику плоский список (отсортирован по дате).
  Поля: `id, title, subject_name, scheduled_at, task_count, corpus, room,
  seat, work_number`.
- **Что показываем:** карточки (название, предмет, дата, корпус, аудитория).
- **Действия:** клик → 3.2.
- **Состояния:** загрузка (скелетон) / пусто / ошибка.
- **Адаптив:** на телефоне — дата + название. ЭКРАН УЧЕНИКА — обязательно
  удобен на телефоне.
- **Образец:** `features/contingent/TeacherContingentPage.tsx` (списки).

### 3.2 Карточка работы (ученик)

- **Маршрут:** `/student/works/:workId`
- **API:** `GET /api/works/{work_id}` — «студенческая» карточка (наблюдатель
  и задания НЕ приходят): `id, title, subject_name, scheduled_at, corpus,
  room, seat, work_number, scores {points, percent, grade}, scans[]`.
  Скан-файл: `GET /api/files/work/{work_id}/scan/{scan_id}`.
- **Что показываем** (3 блока):
  - Основное: название, предмет, дата, корпус, аудитория, место.
  - Сканы (если есть): список файлов, «Скачать» + «Скачать всё» (PDF).
  - Оценки (если есть): баллы по позициям, процент, оценка (2–5, шаг 0.5).
- **Действия:** «Назад», скачивание сканов.
- **Состояния:** загрузка / ошибка; блоки сканы/оценки скрыты, если данных нет.
- **Адаптив:** одна страница, блоки вертикально. ЭКРАН УЧЕНИКА — обязательно
  удобен на телефоне.
- **Образец:** карточка ученика в `features/contingent/TeacherContingentPage.tsx`.

### 3.3 Список работ (учитель/оператор/админ)

- **Маршрут:** `/teacher/works` (также `/operator/works`, `/admin/works`)
- **API:** `GET /api/works/` — роли ≥ учитель приходит готовый объект
  `{ new, old, my }`: `new` = дата ≥ сегодня, `old` = < сегодня, `my` =
  создатель я.
- **Что показываем:**
  - Десктоп: левая панель «Мои работы / Новые / Старые».
  - Редактируемые работы — обводка + «Редактировать»/удалить.
  - Телефон: без панели, общий список.
- **Действия:** переход в карточку (3.4 или 3.6 по правам).
- **Состояния:** загрузка / пусто / ошибка.
- **Адаптив:** на телефоне панель скрыта.
- **Образец:** `features/contingent/TeacherContingentPage.tsx`.

### 3.4 Карточка работы (учитель)

- **Маршрут:** `/teacher/works/:workId`
- **API:** `GET /api/works/{work_id}`. Вход в ученика —
  `GET /api/works/{work_id}/student/{student_id}`.
- **Что показываем:** классруку — корпус + список учеников (ФИО + аудитория);
  наблюдателю — только корпус и аудитория.
- **Действия:** клик по ученику → карточка как у ученика (3.2).
- **Состояния:** загрузка / ошибка / 403.
- **Адаптив:** список вертикально.
- **Образец:** `features/contingent/TeacherContingentPage.tsx`.

### 3.5 Создание работы (модератор)

- **Маршрут:** `/operator/works/new`
- **API:** `POST /api/works/` (WorkCreate): `title, subject_id, student_ids[],
  room_ids[], scheduled_at, observer_ids[]?, task_bank_id?, task_count?,
  test_config_key?, send_notifications (bool), grading_scale[]`.
  Справочники: `GET /api/subjects/`, `/api/classrooms/all`, `/api/tasks/banks`,
  `/api/works/test-configs`.
- **Что показываем:** форма создания.
- **Действия:** заполнить → «Создать» → переход в карточку (3.6).
  Если `send_notifications` включено — после создания показать сообщение
  «Рассадка направлена на почту участникам».
- **Правила** (проверяет бэк, фронт подсказывает): одна параллель на работу;
  учитель — только свой один класс; без банка — `task_count` обязателен.
- **Состояния:** ошибки валидации, загрузка.
- **Адаптив:** поля в один столбец.
- **Образец:** форма `features/settings/SettingsPage.tsx`; выбор участников/
  аудиторий — `features/seating/TeacherSeatingPage.tsx`.

### 3.6 Карточка работы (модератор)

- **Маршрут:** `/operator/works/:workId`
- **API просмотр:** `GET /api/works/{work_id}` — полная карточка: `title,
  subject_name, scheduled_at, task_count, grading_scale, send_notifications,
  creator_id, classes[], rooms[]` (с наблюдателем), `participants[]` (email,
  место, баллы), `scans[]`.
- **Действия:**
  - Редактировать: `PATCH /api/works/{work_id}` (WorkUpdate). Менять можно:
    `title, scheduled_at, test_config_key, task_count` (без банка),
    `student_ids, room_ids`. Остальное (предмет, наблюдатели, банк,
    уведомления) — disabled. Правка участников/аудиторий пересобирает
    рассадку БЕЗ рассылки.
  - Скачать рассадку: `GET .../seating/download` и
    `.../download?sorted_view=true`.
  - Пересоздать рассадку: `POST .../seating/regenerate` (`{student_ids[],
    room_ids[]}`) — без рассылки (показать «уведомления не отправляются»).
  - Зайти в участника: `GET .../student/{student_id}`.
  - Оценки: `POST .../scores` (`{items:[{student_id, points}]}`; `points` =
    `{"позиция": баллы}`). Процент и оценку считает бэк.
- **Что показываем:** классы, участники, аудитории + кнопки.
- **Состояния:** загрузка / ошибка / 403.
- **Адаптив:** блоки вертикально, таблица скроллится.
- **Образец:** `features/contingent/TeacherContingentPage.tsx` + `SettingsPage.tsx`.

### 3.7 Варианты / печать (модератор)

- **Маршрут:** `/operator/works/:workId/variants`
- **API:**
  - Печать вариантов: `POST .../variants/print` (`{student_id}` ИЛИ
    `{room_id}`; нельзя на корпус). Бэк может вернуть warning «рано печатать».
  - Печать ответов: `POST .../variants/answers` (`{copies}`). Только если у
    работы есть банк задач.
  - (Задания берутся из банка на бэке; фронт — кнопка + окно печати.)
- **Что показываем:** выбор «ученик или аудитория» + кол-во комплектов.
- **Действия:** выбрать → окно печати.
- **Состояния:** загрузка / ошибка / предупреждение.
- **Адаптив:** вертикально.
- **Образец:** `features/settings/SettingsPage.tsx`.
