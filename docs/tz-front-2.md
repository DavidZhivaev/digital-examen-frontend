--- A1 Выбор банка --- [ИЮЛЬ: рабочее]
Маршрут: /<роль>/bank
API: GET /api/tasks/banks?subject_id=&parallel=
  Поля: id, title, subject_id, subject_name, parallel, is_global, is_open,
  visibility_percent, positions_count, available_tasks_count, created_by_id.
Что показываем: несколько банков → карточки; один → сразу открыть.
Действия: клик → A2.
Состояния: загрузка / пусто / ошибка.
Адаптив: карточки в один столбец.
Образец: features/contingent/TeacherContingentPage.tsx.

--- A2 Просмотр банка --- [ИЮЛЬ: рабочее]
Маршрут: /<роль>/bank/:bankId
API: GET /api/tasks/banks/{bank_id}?page=&limit=10&q=&position_order=&status=
  Ответ: банк + "mode" + "tasks" (пагинация) + "positions".
  Задача: id, code, position_id, position_order, text (LaTeX), image_*;
  + (если разрешено) solution, answer, author_id; + (normal) status,
  version, latest_review_comment.
  Позиция: id, order, min_score, max_score (≤12); в normal — criteria_text,
  scoring.
Что показываем:
  • public (ученик): задачи по 10 с пагинацией, позиция + текст
    (LaTeX-рендер). Без ответов/поиска.
  • normal: сортировка по позиции, фильтр по позиции/статусу, поиск; мета;
    ответы/решения (админу банка/автору).
Действия: открыть задачу; (normal) поиск/фильтр.
Состояния: загрузка / пусто / ошибка / 403.
Адаптив: список вертикально, фильтры сворачиваются.
Образец: features/contingent/TeacherContingentPage.tsx.
ВАЖНО: текст задач — LaTeX, нужен рендер (KaTeX/react-katex, с лидом).

ЧАСТЬ B. ОБРАБОТКА ЭМ  (всё [ИЮЛЬ: рабочее])

Доступ: загрузка сканов — оператор+; корректура — назначенные (в т.ч.
ученик, если назначен модератором).

--- B1 Загрузка сканов --- [ИЮЛЬ: рабочее]
Маршрут: /operator/processing
API: POST /api/works/{work_id}/scans/upload (multipart: ZIP).
  Внутри ZIP: PDF (имя = номер участника) + results.json.
  Ответ: {work_id, scans_processed, recognition_items_created, warnings[],
  status}.
Что показываем: выбор работы → загрузка ZIP → результат (обработано,
  предупреждения).
Состояния: загрузка / ошибка / предупреждения.
Адаптив: вертикально.
Образец: features/settings/SettingsPage.tsx.

--- B2 Раздел тестовых частей --- [ИЮЛЬ: рабочее]
Маршрут: /operator/processing/test-sections
API: GET /api/works/test-sections — work_id, title, subject_id,
  scheduled_at, status ("Спорные случаи"/"Скачать"), pending_count.
Что показываем: список работ; «Спорные случаи» → корректура (B3);
  «Скачать» → отчёт GET /api/works/{work_id}/recognition/report (Excel).
Состояния: загрузка / пусто / ошибка.
Адаптив: список вертикально.
Образец: features/contingent/TeacherContingentPage.tsx.

--- B3 Корректура распознавания --- [ИЮЛЬ: рабочее]
Маршрут: /operator/processing/:workId/recognition
API пачка: GET /api/works/{work_id}/recognition/batch?limit=5 — до 5
  элементов: id, work_number, position, expected_chars, suggested_text,
  confirmed_text, fragment_url (отрезок скана), status. Бэк закрепляет
  элементы за тобой (двое не проверяют одно).
API подтвердить: POST .../recognition/{item_id}/confirm ({text}).
API назначить помощников (модератор): POST .../recognition/reviewers
  ({user_ids}).
Что показываем: отрезок скана (fragment_url) + поле с текстом + «Верно»
  (или правка → «Верно»). Пачка кончилась → новая.
Состояния: загрузка / пусто («всё проверено») / ошибка.
Адаптив: фрагмент + поле вертикально.
Образец: интерактив features/seating/TeacherSeatingPage.tsx.
