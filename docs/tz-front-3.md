ЧАСТЬ B. АУДИТОРИИ
================================================================
Доступ: просмотр одной — любой; список и управление — оператор+.

--- B1 Список аудиторий ---
Маршрут: /operator/classrooms (и /admin/classrooms)
API: GET /api/classrooms/all?page=&page_size= — RoomResponse: id, corpus,
  number, rows, columns, it, seats_count.
Что показываем: таблица («корпус-номер», места rows×columns, бейдж IT
  если it=true), пагинация.
Действия: создать (B2), клик → редактировать (B2), удалить.
Состояния: загрузка / пусто / ошибка.
Адаптив: таблица скроллится.
Образец: features/contingent/TeacherContingentPage.tsx (таблица).

--- B2 Создание / редактирование аудитории ---
Маршрут: модалка в B1
API:
  Создать: POST /api/classrooms/create (corpus 1-4, number, rows 1-20,
   columns 1-20, it bool).
  Изменить: PATCH /api/classrooms/{room_id}.
  Удалить: DELETE /api/classrooms/{room_id}.
Что показываем: форма — корпус, номер, строки и столбцы (= места), IT.
Действия: заполнить → сохранить.
Состояния: ошибки валидации / загрузка.
Адаптив: поля в один столбец.
Образец: features/settings/SettingsPage.tsx (форма).

ЧАСТЬ A. КОНТИНГЕНТ
================================================================

--- A1 Список классов ---
Маршрут: /teacher/classes (и /operator/users, /admin/users)
API кратко: GET /api/classes/show-short — учитель видит свои классы;
  оператор+ — все. ClassResponse: id, teacher_id, parallel, litera,
  corpus, display_name, students_count, students_group_first[],
  students_group_second[], teachers_group_first[], teachers_group_second[],
  history[].
API полно: GET /api/classes/show-full (с email/login).
Что показываем: учитель с одним классом → сразу A2; несколько/оператор+ →
  выбор параллель → класс (быстро, с возвратом) → A2.
Действия: выбор класса.
Состояния: загрузка / пусто / ошибка.
Адаптив: выбор вертикально.
Образец: features/contingent/TeacherContingentPage.tsx (готов, НЕ трогать).

--- A2 Класс: ученики и учителя ---
Маршрут: /teacher/classes/:classId
API: GET /api/classes/{class_id}/full — класс, классрук (teacher),
  ученики по группам (id, person_id, ФИО, email, login, group),
  предметники (teachers: id, ФИО, subject, groups), history.
Ученики: GET /api/classes/{class_id}/students (id, person_id, email,
  login, ФИО, group, must_set_password, role). must_set_password=true →
  «не заходил».
Что показываем: таблица учеников (ФИО, почта, группа, статус),
  предметники, классрук. Плашка-проверка «всё ли заполнено» (скрываемая).
Действия: клик по ученику → A3; кнопки управления (классрук/оператор+) — A4.
Состояния: загрузка / пусто / ошибка / 403.
Адаптив: таблица скроллится, табы по группам.
Образец: features/contingent/TeacherContingentPage.tsx (готов, НЕ трогать).

--- A3 Действия с учеником ---
Маршрут: модалка/панель в A2
API:
  Создать: POST /api/users/ (email, role=1, class (class_id), class_group,
   ФИО, sex?). Новый помечен must_set_password=true.
  Добавить существующего: POST /api/classes/{class_id}/students
   ({user_id, group}).
  Сменить группу: PATCH .../students/{user_id}/group ({group}).
  Перевести к себе: POST .../students/{user_id}/transfer ({target_class_id,
   group}). UI: поиск по ФИО → выбор (с текущим классом) → группа →
   подтвердить.
  Удалить из класса: DELETE .../students/{user_id}.
  Сменить пароль: POST /api/users/{person_id}/force-password-reset (оператор+).
Что показываем: карточка ученика (все поля, вкл. id), кнопки действий.
Состояния: загрузка / ошибки / 403.
Адаптив: формы вертикально.
Образец: карточка в features/contingent/TeacherContingentPage.tsx.

--- A4 Управление классом ---
Маршрут: модалки в A2
API:
  Создать класс: POST /api/classes/ (parallel 1-12, litera, corpus,
   teacher_id?).
  Изменить: PATCH /api/classes/{class_id} (parallel?, litera?, corpus?) —
   классрук/оператор+.
  Удалить: DELETE /api/classes/{class_id}?delete_students= — оператор+.
  Назначить классрука: PATCH .../teacher ({teacher_id}) — оператор+.
  Снять классрука: DELETE .../teacher — оператор+.
  Назначить предметника: POST .../assign-teacher ({teacher_id, group?
   (null=весь класс), subject}). UI: выбрать предмет (GET /api/subjects/)
   → найти учителя ЭТОГО предмета по ФИО → группа → сохранить.
  Снять предметника: DELETE /api/classes/teacher-assignment/{assignment_id}
   или совсем: DELETE .../teacher/{teacher_id}.
  Экспорт класса: GET .../export?group= (xlsx).
Что показываем: модалки класса, назначения учителей, кнопка экспорт.
Состояния: загрузка / ошибки / 403.
Адаптив: модалки на весь экран на телефоне.
Образец: SettingsPage.tsx (формы), TeacherSeatingPage.tsx (поиск).
ПРИМЕЧАНИЕ: импорт класса из Excel — ОСЕНЬ (Этап 2). К июлю ученики
  вносятся вручную.

--- A5 Выборка всех пользователей ---
Маршрут: /operator/users/all (и /admin/users/all)
Доступ: учитель и выше (по коду).
API: GET /api/users/?page=&page_size= — PaginatedUsersResponse: items[] +
  total, page, page_size, pages. UserResponse: id, person_id, email, login,
  role, register_at, class_id, class_group, must_set_password, ФИО, sex,
  email_accept, last_do. Справочник ролей: GET /api/users/roles.
Что показываем: таблица всех пользователей с пагинацией, фильтр по роли.
  Кнопка «Создать персону»: POST /api/users/ (роль ученик → выбрать класс
  и группу).
Действия: клик → карточка (A6).
Состояния: загрузка / пусто / ошибка / 403.
Адаптив: таблица скроллится.
Образец: features/contingent/TeacherContingentPage.tsx.

--- A6 Карточка пользователя ---
Маршрут: модалка из A5
API:
  Один: GET /api/users/{person_id}; Изменить: PATCH /api/users/{person_id}.
  Сменить роль: PATCH .../role ({role}) — админ.
  Форс-сброс пароля: POST .../force-password-reset.
  Закрыть все сессии: DELETE /api/auth/sessions/user/{user_id}.
  Удалить: DELETE /api/users/{person_id} — админ.
Что показываем: данные пользователя, кнопки изменить/роль/пароль/сессии/
  удалить. Если учитель — классы, где ведёт, с возможностью снять.
Состояния: загрузка / ошибки / 403.
Адаптив: карточка вертикально.
Образец: features/contingent/TeacherContingentPage.tsx + SettingsPage.tsx.
