# ROADMAP — план работ и проверок

Как лид проверяет работу:
1. ПЕРВАЯ ПРОВЕРКА — через 2-3 дня после старта.
2. Дальше — ЕЖЕНЕДЕЛЬНО, по понедельникам-вторникам.

ВАЖНО: если намечаются нюансы, из-за которых можешь не успеть к
контрольной точке (заболел, экзамены, застрял, уезжаешь) — сообщи лиду
СРАЗУ, не дожидаясь проверки. Лучше предупредить заранее, чем сорвать
срок молча.

Каждый экран сдаётся отдельным PR. Экран готов = состояния (загрузка/
пусто/ошибка) + адаптив + `npm run build` проходит + PR прошёл ревью.

(Коды в скобках — для сверки с твоим ТЗ.)

================================================================
ПРОВЕРКА 1 — через 2-3 дня после старта (разминка)
================================================================
К этому моменту от каждого: запущен проект, прочитаны ТЗ + FRONTEND_GUIDE,
открыт ПЕРВЫЙ PR с простым экраном (на моках):

  Фронт 1:  Список работ ученика (3.1)
  Фронт 2:  Выбор банка задач (A1)
  Фронт 3:  Список аудиторий (B1)

Цель — убедиться, что человек разобрался с проектом и реально начал.

================================================================
ПОНЕДЕЛЬНИК 1 (конец 1-й недели)
================================================================
  Фронт 1:  Список работ ученика + Карточка работы ученика
  Фронт 2:  Выбор банка задач + Загрузка сканов
  Фронт 3:  Список аудиторий + Создание аудитории + Список классов
  Итого у каждого: ~2-3 экрана сданы.

================================================================
ПОНЕДЕЛЬНИК 2 (конец 2-й недели)
================================================================
  Фронт 1:  Список работ учителя + Карточка работы учителя
  Фронт 2:  Просмотр банка задач (LaTeX) + Раздел тестовых частей
  Фронт 3:  Класс: ученики и учителя + Действия с учеником
  + С этой недели начинается интеграция с реальным API —
    ТОЛЬКО по сигналу лида (когда готова авторизация).

================================================================
ПОНЕДЕЛЬНИК 3 (конец 3-й недели) — самые сложные экраны
================================================================
  Фронт 1:  Создание работы (большой, сложный)
  Фронт 2:  Корректура распознавания (сложный)
  Фронт 3:  Управление классом
  Если сложный экран не пошёл — сообщить лиду заранее.

================================================================
ПОНЕДЕЛЬНИК 4 (конец 4-й недели) — сквозной сценарий
================================================================
  Фронт 1:  Карточка работы (модератор) + Варианты/печать
  Фронт 2:  Конструктор задачи (только ВИД, на моках) + доделки
  Фронт 3:  Выборка всех пользователей + Карточка пользователя
  К этому моменту должен проходить СКВОЗНОЙ сценарий: создал работу →
  рассадка → сканы → корректура → оценки → ученик видит результат.

================================================================
ПОНЕДЕЛЬНИК 5 (конец 5-й недели) — сдача
================================================================
  Все: выставление оценок, багфиксы, адаптив, полировка.
  Всё рабочее, задеплоено, сдано.

================================================================
ЧТО НЕ ДЕЛАЕМ К ИЮЛЮ (Этап 2, осень)
================================================================
  - Банк задач: конструктор LaTeX (рабочий), модерация, версии,
    управление, оцифровка. К июлю — только просмотр + вид конструктора.
  - Аналитика.
  - Импорт класса из Excel (к июлю — ввод вручную).
  - Модуль абитуриентов, зачёты.
